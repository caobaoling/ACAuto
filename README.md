# 51AirClass 性能测试自动化工具

自动完成 51AirClass 设备检测中的性能测试流程：识别并点击「开始测试」→ 等待测试结束 → 自动截图保存。

---

## 环境要求

- Python 3.10+
- Windows 系统（脚本使用屏幕截图和鼠标点击）

---

## 安装依赖

```bash
pip install -r requirements.txt
```

### 依赖说明

| 包 | 版本 | 用途 |
|----|------|------|
| easyocr | >=1.7.2 | OCR 识别屏幕文字（中文+英文） |
| pyautogui | >=0.9.54 | 截取屏幕、模拟鼠标点击 |
| Pillow | >=11.3.0 | 图像处理与截图保存 |
| numpy | >=2.4.4 | 图像数组转换（easyocr 依赖） |

> **注意**：easyocr 依赖 PyTorch，安装时会自动拉取。首次运行会下载中文 OCR 模型（约 300MB），需要网络连接，之后模型缓存在本地无需重复下载。

---

**1. 打开 51AirClass，进入设备检测的「性能」页面**

确保页面右下角可见「开始测试」按钮，如下图所示：

```

│                             │
│                  [开始测试] │
```

**2. 运行脚本**

```bash
python D:\ACAuto\auto_test.py
```

## 可调参数

编辑 `auto_test.py` 顶部的配置区域：

```python
SAVE_DIR       = r"D:\ACAuto\results"  # 截图保存目录
POLL_INTERVAL  = 3                      # 轮询间隔（秒），越小越灵敏但 CPU 占用更高
TIMEOUT        = 600                    # 最长等待时间（秒），超时自动退出
CONFIDENCE     = 0.6                    # OCR 置信度阈值，越高越严格
```

## 常见问题

**Q：提示"未找到「开始测试」按钮"**
- 确认 51AirClass 已打开并切换到「性能」标签页
- 确认按钮在屏幕可见区域内（未被遮挡）
- 适当降低 `CONFIDENCE` 值（如改为 `0.5`）后重试

**Q：首次运行很慢**
- easyocr 首次运行需下载中文模型（约 300MB），下载完成后速度正常

**Q：Step 2 提示"未检测到停止测试"**
- 不影响后续流程，脚本会继续轮询等待"测试完成"

**Q：超时退出（600秒）**
- 增大 `TIMEOUT` 值后重试，或检查 51AirClass 是否正常运行
